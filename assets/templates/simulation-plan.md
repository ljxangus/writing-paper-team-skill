## 1. 实验分类

### 1.1 实验类型（选择一项）

- [ ] **网络包级仿真（Network Packet-Level Simulation）**
  - 描述：对网络拓扑中单个报文的传输进行逐包建模
  - 典型工具：simpy, ns-3, OMNeT++
  - 关键指标：吞吐量、端到端时延、丢包率、抖动
  - 必须实现：报文生成 → 排队 → 传输（含传播时延） → 接收，每个报文有唯一 ID

- [ ] **网络流级仿真（Network Flow-Level Simulation）**
  - 描述：对聚合流量的到达、传输、完成进行建模
  - 典型工具：Python (numpy/scipy), MATLAB
  - 关键指标：流完成时间、聚合吞吐量、公平性指数
  - 必须实现：流到达过程（如 Poisson）、流大小分布、带宽共享模型

- [ ] **排队/资源分配仿真（Queuing / Resource Allocation Simulation）**
  - 描述：对资源竞争、调度策略、排队行为进行建模
  - 典型工具：simpy, 自定义 DES
  - 关键指标：队列长度、等待时间、利用率、拒绝率
  - 必须实现：作业到达 → 服务队列 → 调度策略 → 作业完成

- [ ] **机器学习实验（Machine Learning Experiment）**
  - 描述：在标准数据集上训练和评估 ML 模型
  - 典型工具：PyTorch, TensorFlow, scikit-learn
  - 关键指标：Accuracy, F1, AUC, 训练时间, 推理延迟
  - 必须实现：完整训练循环（前向 → 损失 → 反向 → 更新），多 epoch，train/val/test 划分

- [ ] **分布式系统仿真（Distributed Systems Simulation）**
  - 描述：对分布式共识、复制、容错进行建模
  - 典型工具：simpy, 自定义事件驱动模型
  - 关键指标：共识延迟、消息复杂度、故障恢复时间
  - 必须实现：节点间消息传递、故障模型、协议状态机

- [ ] **解析+数值混合（Analytical + Numerical Hybrid）**
  - 描述：结合解析模型和数值计算的性能评估
  - 典型工具：numpy, scipy, MATLAB
  - 关键指标：收敛速度、计算开销、与理论值的偏差
  - 注意：必须明确声明哪些结果是解析计算、哪些来自数值仿真

- [ ] **其他：[在此说明]**

### 1.2 仿真范式（选择一项）

- [ ] **离散事件仿真（DES）**
  - 定义：系统状态在离散时间点由事件驱动改变
  - 必须证据：事件日志，格式 `{timestamp, event_type, entity_id, state_before, state_after}`

- [ ] **蒙特卡洛仿真（Monte Carlo）**
  - 定义：通过重复随机采样获取数值结果
  - 必须证据：逐次迭代的采样数据、收敛曲线数据

- [ ] **时间步进仿真（Time-Stepped）**
  - 定义：系统状态以固定时间步长推进
  - 必须证据：每个时间步的状态快照

- [ ] **基准评估（Benchmark Evaluation）**
  - 定义：在标准基准数据集上按固定协议训练和评估
  - 必须证据：训练曲线（每 epoch）、评估输出

- [ ] **仿真器/测试床（Emulation / Testbed）**
  - 定义：在受控环境中运行实际系统组件
  - 必须证据：系统日志、资源监控、报文捕获
